2.1 Systemanforderungen

Server

  • Web server
  • PHP Version höher 4.3.0
  • PostgreSQL höher als 7.4 oder MySQL höher als 4.0

PostgreSQL ist empfohlen, da es eine bessere Unterstützung für Transaktionen hat. Wenn Sie PostgreSQL benutzen, wird die Verwendung von 7.4.x oder 8.0.x empfohlen.

Wenn Sie MySQL benutzen, wird die Verwendung von 4.1.x empfohlen (Das ist die einzige Version mit der ich getestet habe) [Anmerkung des Übersetzers: Mein Bug Tracker läuft einwandfrei mit MySQL 5.0]. Sie sollten MySQL mit Unterstützung für den UTF8-Zeichensatz kompilieren; damit werden bessere Ergebnisse bei der Sortierung erzielt. Wenn Sie Dateien mit mehr als 1MB Größe hochladen wollen, müssen sie (neben Einschränkungen bei PHP - hier beträgt die Standardeinstellung 2MB) die Einstellung für das "Communication Packet" von MySQL ändern. Ein "Communication Packet" ist ein einzelnes SQL-Statement, das an den MySQL-Server gesendet wird. Die Standardeinstellung beträgt 1 MB. Sie können diese Einstellung ändern indem Sie die Option --max_allowed_packet=16M beim Start des mysqld hinzufügen.

Client

  • Webbrowser mit Unterstützung für JAVA-Script (Einige Funktionen können mit Unterstützung für JAVA Script deaktiviert werden)

2.2 Bug Tracker Konfiguration

Es ist sehr einfach Bug Tracker zu installieren. Folgende Schritte sind notwendig:

  • Laden Sie Bug Tracker herunter
  • Entpacken Sie Bug Tracker in das Wurzelverzeichnis ihres Webspace
  • Ändern Sie die Datei "bug/include/config.php"
  • Verbinden Sie sich auf http://your.ip.com/bug/setup/index.php und folgen Sie dem Assistenten.

Nach dem Entpacken von Bug Tracker müssen Sie die Datei include/config.php ändern , bevor Sie sich zum Installationsassistenten verbinden:

<?
$GLOBALS['BR_dbserver'] = "127.0.0.1"; // IP-Adresse des Datenbankservers
$GLOBALS['BR_dbuser'] = "pgsql"; // Benutzername zum Zugriff auf die Datenbank
$GLOBALS['BR_dbpwd'] = "password"; // Datenbankpasswort
$GLOBALS['BR_dbname'] = "bugdb"; // Name der Datenbank für Bug Tracker

/* Real path of bug tracker project */
$GLOBALS["SYS_PROJECT_PATH"] = "/usr/local/apache/htdocs/bug";

/* The path of URL. If you have to connect to the Bug Tracker by
 * http://192.168.0.1/bug/index.php, you should input "/bug" here.
 */
$GLOBALS["SYS_URL_ROOT"] = "/bug";
?>

Nachdem Sie die Datei angepasst haben, können Sie sich mit ihrem Webbrowser zum Installationsassistenten setup/index.php über die URL http://your.ip.com/bug/setup/index.php verbinden. Beachten Sie das "/bug/setup/index.php" von ihrem Bug Tracker Pfad abhängig ist.

Jetzt müssen Sie nur noch den Anweisungen auf dem Bildschirm folgen um die Installation abzuschliessen. Nachdem das beendet ist, können Sie sich über die URL http://your.ip.com/bug/index.php zu Ihrem Bug Tracker verbinden. Der Benutzername des Administrators ist "admin" und das voreingestellte Passwort ist ebenfalls "admin". Sie können (und sollten) das Passwort nach dem Anmelden ändern.

Das ist Alles. Einfach, oder?

Zur Beachtung: Nachdem Sie die Installation beendet haben, entfernen Sie bitte das "setup/*.php" - Verzeichnis wenn Sie nicht noch weitere Sprachen hinzufügen wollen. Um Ihre Sprache hinzuzufügen, beachten Sie bitte Kapitel 5.

2.3 Kundenfeedbacksystem konfigurieren

Sie können dieses Kapitel überspringen, wenn Sie keine Oberfläche für das Kundenfeedback brauchen. Um das Kunden- oder ein Anonymes Feedbacksystem zu konfigurieren, müssen Sie nur die Datei feedback_site/include/config.php anpassen um die Datenbankinformationen zu konfigurieren. Danach können Sie sich über die URL http://your.ip.com/bug/feedback_site/index.php zum Feedbacksystem verbinden. Sie können neue Kunden hinzufügen, oder auch anonymen Zugriff erlauben. Für weitere Informationen lesen Sie bitte Kapitel 4.3 .

Das Feedbacksystem kann völlig vom Bug Tracker getrennt werden. Sie können das Verzeichnis "feedback_site" auf einen anderen Webserver oder in ein anderes Verzeichnis verschieben, wenn Sie wollen. Zum Beispiel können Sie das "feedback_site" - Verzeichnis ins Wurzelverzeichnis ihres Webspace verschieben, und in "feedback" umbenennen. Danach können Sie auf das Feedbacksystem über die URL http://your.ip.com/feedback/index.php zugreifen.

2.4 Upgrade von älterer Version

Wenn Sie von einer älteren Version upgraden, führen Sie bitte folgende Schritte durch:

  • Laden Sie die aktuelle Version von Bug Tracker herunter
  • Entpacken Sie sie ins Wurzelverzeichnis ihres Webspace
  • Ändern Sie die Datei "bug/include/config.php" entsprechend
  • Verbinden Sie sich auf die URL http://your.ip.com/bug/setup/upgrade.php und führen Sie den Assistenten aus.
